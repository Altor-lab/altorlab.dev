altor-vec benchmark comparisons

These pages compare altor-vec with server-side vector databases, database-adjacent tools, and browser-capable libraries. The comparisons are intentionally honest: backend systems win on scale and operations, while altor-vec wins when local UX, bundle size, and zero per-query cost matter more.

ChromaDBApplication database for embeddings versus static browser retrieval.ChromaDB is better as an application data layer. altor-vec is better as a shipped frontend retrieval primitive.
LanceColumnar/vector data platform tradeoffs versus browser-native ANN.Lance is a data platform choice. altor-vec is a frontend feature choice.
Milvus LiteEmbedded/server vector database tradeoffs versus fully in-browser retrieval.Milvus Lite is the better stepping stone to a vector backend. altor-vec is the better fit for pure frontend retrieval.
